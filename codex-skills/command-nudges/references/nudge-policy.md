# Nudge Policy

Good nudges:

- After a new durable preference:
  `Useful next command: save memory if you want this preference preserved.`

- Before a large technical plan:
  `Relevant skill: survey before we lock the architecture.`

- After choosing a non-obvious tradeoff:
  `Useful next command: decision log so future-us remembers why.`

- After a failure:
  `Relevant skill: post-mortem if this should never repeat.`

Bad nudges:

- Every answer.
- Multiple commands at once.
- Reminders after trivial chat.
- Robotic phrasing like "Please execute command X."

Default to silence unless the nudge clearly improves future results.
